# YEZI 开发文档

## 技术栈

- Next.js App Router
- React 19
- SQLite：使用 Node.js 24 内置 `node:sqlite`
- 样式：全局 CSS，视觉参考截图中的极简等宽大写风格

## 本地运行

```bash
npm install
npm run dev
```

默认打开 `http://localhost:3000`。如果端口被占用，Next.js 会自动切换到下一个可用端口。

## 目录结构

```text
app/                 Next.js 页面与 API Route
components/          可复用 React 组件
lib/                 SQLite、认证、订单、支付、种子商品数据
public/products/     商品图片资源
public/uploads/      后台上传图片
doc/                 功能说明和开发文档
data/                SQLite 数据库，运行后自动生成
scripts/             辅助脚本
```

## 数据库

数据库文件：`data/yezi.db`

表结构定义：`lib/schema.sql`

主要表：

- `products`：商品主数据。
- `product_variants`：SKU 变体、尺码、库存、预留库存。
- `cart_items`：访客购物车行。
- `users`：用户账户。
- `user_sessions`：登录会话。
- `addresses`：用户地址簿。
- `orders`：订单主表。
- `order_items`：订单商品行。
- `inventory_movements`：库存流水。
- `payments`：支付记录。
- `site_settings`：站点配置预留。

应用首次访问时会自动执行 schema，并在 `products` 为空时写入 `lib/products.ts` 中的种子商品。旧库会通过 `lib/db.ts` 中的轻量迁移补齐新列，并自动补齐变体和默认管理员。

默认管理员：

```text
admin@yezi.local
admin123
```

## 商品数据维护

当前种子商品数据在 `lib/products.ts` 中维护。后台也可以在 `/admin` 创建新商品。

每个商品包含：

- `sku`：展示编号。
- `slug`：详情页路径。
- `category`：分类筛选值。
- `priceCents`：价格，单位为美分。
- `image`：列表图。
- `gallery`：详情轮播图。
- `stock`：库存数。
- `material`：材质说明。
- `sizeChart`：尺码表。
- `careInstructions`：护理说明。
- `modelInfo`：模特信息。

修改种子数据后，如果要重建本地数据库，可删除 `data/yezi.db` 后重新启动应用。

## 商品图片

当前图片从用户提供的截图裁切生成，放在 `public/products/`。

后台上传图片会写入 `public/uploads/`，并返回可用于商品 `image` 或 `gallery` 的 URL。

辅助脚本：

```bash
node scripts/crop-assets.mjs
```

## 订单与库存

下单流程：

1. 读取当前会话购物车。
2. 校验变体可售库存：`stock - reserved`。
3. 创建订单，状态为 `inventory_locked`。
4. 写入订单行。
5. 增加变体 `reserved` 数量，并写入 `inventory_movements`。
6. 创建支付记录。
7. 开发模拟支付会立即把订单标记为 `paid`，并从 `stock` 中扣减库存、释放 `reserved`。

## 支付配置

当前支付入口支持三种 provider：

- `stripe`
- `paypal`
- `apple_pay`

如果没有配置真实密钥，系统使用开发模拟支付。

预留环境变量：

```bash
STRIPE_SECRET_KEY=
PAYPAL_CLIENT_ID=
PAYPAL_SECRET=
NEXT_PUBLIC_APPLE_PAY_ENABLED=true
```

后续接真实支付时，需要补充 provider SDK、支付回调、Webhook 签名校验、失败释放库存和订单状态变更。
